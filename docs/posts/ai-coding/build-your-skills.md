## Custom skills 非常重要

开源skills 他不是 npm包 使用起来存在很多问题

{一个表格对比 开源skills npm包 customSkills}

skill 不是一个好用的黑盒

skill 永远处在长期优化的过程中，即使是‘帮我生成代码注释’，也没有适配所有人的skills，也没有最终答案的skills

所以全盘了解你的skill，是必然，自己从0 custom skill，或是最佳

## skill的本质是对工作流的收束

## 及时输出文档

我说的这个文档不是项目文档，而是

Vibe Coding 的过程中时常让AI输出文档进行记录，这些文档中有很多待发掘的 "惊喜点"

Vibe Coding 过程中我们的描述往往 包含两类 1简单 抽象的，2我们想到的必要的要求

其中1保证了 AI编写文档时的发散性，其中经常会出现很好用的写法或展示思路，或是一个表格列，或是一个文字图，你知道这个输出很好用，一目了然

2则保证了文档中 含有我们日常临时想到的一些规则，

积累并及时总结这些 AI生成的这些 发散文档中的你喜欢的点 与 你临时想到的规则 进行收束，生成skill

如果临时想生成一个skill，就像限时写作文，只会觉得难受，‘有这时间我都vibe coding出来代码了’，于是skill永远搁置，生产力与团队规范留到下一次再说

## 及时记录提示词

Vibe Coding 过程中 2我们想到的必要的要求，这都是对这一工作流的收束

## skills的价值在哪里

在于 在具体某一工作中 【约束 ai 的思路】【输出我们想要的规范的内容】【一次性得到结果】
